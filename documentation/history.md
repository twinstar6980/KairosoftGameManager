## 24-01-09

Windows 1.0

## 24-02-01

Windows 1.1

* 修复了在 Windows 10 系统中无法选择目录的 BUG 。

* 存档加密、解密功能中的 Key 参数从 64 位无符号整数更改为任意长度字节数组，以支持 Android 与 iPhone 版本的开罗游戏存档。

## 24-03-21

Windows 1.2

* 测试了部分开罗游戏，将其版本信息加入信任名单中。

* 修改 Modify Program 功能中 Enable Debug Mode 的默认值为 true 。

* 修改 Modify Program 功能中备份文件的路径，以避免游戏版本更新后出现文件混乱的问题。

* 使部分文本控件能够进行选择交互。

## 24-05-23

Windows 1.3

* 迁移项目至 Visual Studio 2022 17.10 。

* 修改应用缓存目录位置。

* 游戏列表项的标题文本不再占据整行空间，以此避免鼠标移动至空白处却仍显示文本选择光标的问题。

* 优化代码。

Windows 4.0

## 25-10-15

Windows 5.0

* 弃用了之前使用的 PowerShell 构建脚本，现在通过 Python 脚本进行构建。

* 更改了程序备份的路径；在通过功能页的直接修改指定的程序时，将不再对程序原文件进行备份。

* 功能页中添加了指示运行状态的进度条，并且每次运行功能时会清空先前的消息文本。

* 修改程序功能的启用调试模式参数默认值更改为 `false` 。

* 修改程序功能现在可以直接处理 `zip` 、`apk` 、`apks` 安装包格式的文件。

* 修改程序功能现在可以正确处理游戏目录内同时存在多种架构的程序文件的情况。

* 功能页中完成任务后将弹出消息通知。

## 25-10-29

Windows 7.0

* 更改应用文件存储路径。

* 调用外部程序时，若指定的程序路径文件不存在，将尝试检索 `PATH` 环境变量中的对应程序。

* 修改程序功能现在将确保修改成功后再对 zip / apk / apks 应用包文件进行修改；此外，原文件将被移至回收站而非直接删除。

* 修改程序功能现在将能够对 apk / apks 应用包文件进行对齐与签名，需要用户配置相应设置项，若设置项无效，将跳过对齐与签名步骤。

* 管理页中，若指定的 Steam 库路径无效，将出现警告而非报错。
